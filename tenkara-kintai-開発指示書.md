# 飛騨牛食べ処てんから 勤怠・給与計算システム 開発指示書

## 概要

飲食店「飛騨牛食べ処てんから」向けの勤怠打刻・給与計算Webアプリケーションを開発する。
スタッフがタブレットで出退勤を打刻し、管理者がPCで勤務時間の確認・給与計算・明細出力を行う。
最終的に「差引支給額を封筒に入れて渡せばよい」状態を実現する。

## 画面構成と機能仕様

### 1. 打刻画面（タブレット用 /punch）

**目的**: パートスタッフが自分で出退勤を記録する画面

**UI仕様**:
- 画面上部に現在時刻を大きく表示（リアルタイム更新）
- スタッフ名の一覧をグリッド表示（顔写真なし、名前ボタンのみ）
- スタッフ名をタップ → 4つのボタンを表示
  - 出勤（緑）
  - 退勤（赤）
  - 休憩開始（黄）
  - 休憩終了（青）
- 現在の状態（出勤中/休憩中/退勤済み/未出勤）をスタッフ名の横に表示
- 打刻成功時にフィードバック表示（「○○さん、出勤しました 9:00」など）
- 誤打刻防止: 出勤していないのに退勤ボタンを押す等の場合は警告

**状態遷移ルール**:
- 未出勤 → 出勤のみ可能
- 出勤中 → 退勤 or 休憩開始が可能
- 休憩中 → 休憩終了のみ可能
- 休憩終了後 → 退勤 or 再度休憩開始が可能
- 退勤済み → 再出勤可能（同日に複数回出勤するケースに対応）

### 2. 管理ダッシュボード（PC用 /admin）

**認証**: 管理者パスワードで保護（初期設定画面で設定）

**ダッシュボード内容**:
- 当日の出勤状況一覧（出勤中/休憩中/退勤済み/未出勤）
- **タイムライン表示**: 横軸=時間（6:00〜翌2:00）、縦軸=全スタッフ名
  - 勤務中: 青系のバー
  - 休憩中: グレーまたは斜線
  - 深夜時間帯（22:00以降）: バーの色を変える or 背景に薄い色
  - 現在時刻に縦線
  - 当日だけでなく日付選択で過去の日も確認可能

### 3. スタッフ管理（/admin/staff）

**スタッフ一覧画面**:
- 全スタッフの一覧表示（名前、雇用形態、給与形態、時給/基本給、状態）
- 新規登録ボタン
- 各スタッフの編集・無効化ボタン

**スタッフ登録/編集画面**:
- 基本情報: 氏名、肩書（任意）
- 雇用形態: 正社員 / パート（ラジオボタン）
- 給与形態: 月給制 / 時給制（スイッチ切り替え）
  - 月給制の場合: 基本給（月額）入力欄 + 時給欄（残業計算用単価）
  - 時給制の場合: 時給入力欄
- 税区分: 甲欄 / 乙欄
- 雇用保険: 対象 / 対象外
- 健康保険料: 月額（0で対象外、社員のみ想定）
- 介護保険料: 月額（0で対象外）
- 厚生年金: 月額（0で対象外）
- 食事代控除: 対象 / 対象外
- 家賃控除: 月額（0で控除なし）
- 交通費: あり / なし
- 従業員ページパスワード: 設定・変更

### 4. 勤怠一覧・修正（/admin/attendance）

**月次勤怠一覧**:
- 年月選択 + スタッフ選択（全員 or 個別）
- 日別の出勤時刻・退勤時刻・休憩時間・実労働時間を一覧表示
- **タイムライン表示**: スタッフ別に月間の勤務パターンを視覚化

**打刻修正機能**:
- 管理者のみ操作可能
- 打刻データの追加・編集・削除
- 修正時にis_modifiedフラグをtrueに設定

### 5. 給与計算（/admin/payroll）

**操作フロー**:
1. 対象年月を選択
2. 月次手動入力の確認・編集（食数など）
3. 「給与計算実行」ボタンで全スタッフ一括計算
4. 計算結果一覧を表示
5. 個別スタッフの明細を確認
6. 「確定」ボタンで保存

**計算結果表示**:
- 全スタッフの一覧: 名前、支給総額、控除総額、差引支給額
- **現金封入用表示**: 差引支給額を大きなフォントで表示、紙幣・硬貨の枚数内訳も表示
- 個別明細: 後述の給与明細フォーマットと同等の内容

## 計算ロジック詳細

### 勤務時間の集計

**1日の勤務時間計算**:
```
実労働時間 = (退勤時刻 - 出勤時刻) - 休憩時間合計
休憩時間 = Σ(休憩終了 - 休憩開始)
```
- 1分単位で計算（端数切り捨てなし）
- 日またぎ勤務なし

**時間区分の振り分け**（1日単位）:
```
通常時間 = min(実労働時間, 480分)  ※8時間 = 480分
残業時間 = max(実労働時間 - 480分, 0)
深夜時間 = 22:00〜翌5:00にかかる実労働時間（休憩除く）
```

**休日判定**（週単位）:
- 週の起算日: 水曜日（水曜〜翌火曜が1週間）
- その週の6日目以降の勤務は休日扱い（×1.25）
- 勤務日のカウントは出勤日の時系列順

**割増率の適用**:
- 通常: ×1.0
- 残業（8時間超）: ×1.25
- 深夜（22:00以降）: ×1.25
- 休日（週6日目以降）: ×1.25
- 重複時: 1.25倍のまま（加算しない）
  - 例: 残業かつ深夜 → 1.25倍（1.5倍にはならない）
  - 例: 休日かつ深夜 → 1.25倍

### 支給額計算

**時給制スタッフ**:
```
通常賃金 = 時給 × 通常時間
残業手当 = 時給 × 1.25 × 残業時間（深夜・休日以外の残業）
深夜手当 = 時給 × 1.25 × 深夜時間
休日手当 = 時給 × 1.25 × 休日時間
```

**月給制スタッフ**:
```
基本給 = 月額固定
残業単価 = 時給欄の金額
残業手当 = 残業単価 × 1.25 × 残業時間
深夜手当 = 残業単価 × 1.25 × 深夜時間
休日手当 = 残業単価 × 1.25 × 休日時間
```

**月給制の欠勤控除（ノーワーク・ノーペイ）**:
```
月の所定労働日数 = その月の営業日数（土日祝を除く平日数、ただし飲食店のため実態に合わせて手動設定も可）
※ 簡易実装: 管理画面で月ごとの所定労働日数を設定できる欄を設ける（デフォルト: 暦日数 - 日曜数くらいの概算 or 固定値）
1日あたり単価 = 基本給 ÷ 月の所定労働日数
欠勤控除 = 1日あたり単価 × 欠勤日数
遅刻・早退控除 = (基本給 ÷ 月の所定労働時間) × 不足時間
```
※ 所定労働日数の設定は管理画面の「月次設定」に欄を設ける

**共通手当**:
```
交通費 = 出勤日数 × 500円（対象者のみ）
まかない手当 = 食数 × 125円
```

**支給総額**:
```
支給総額 = 基本給（or 通常賃金）+ 残業手当 + 深夜手当 + 休日手当 + 交通費 + まかない手当 - 欠勤控除（月給制のみ）
```

### 控除額計算

**所得税**:
- 令和7年分の源泉徴収税額表（月額表）を使用
- 甲欄: 扶養人数0人の列を適用（扶養人数の管理は今回スコープ外。必要になれば追加）
- 乙欄: 乙欄の列を適用
- 課税対象額 = 支給総額 - 交通費 - 社会保険料合計（健康保険 + 介護保険 + 厚生年金 + 雇用保険）
- 税額表から該当する金額帯の税額を返す

**所得税の税額表データ**:
- 国税庁「令和7年分 源泉徴収税額表（月額表）」をJSONデータとしてシステムに組み込む
- 税額表の更新は年1回、JSONファイルの差し替えで対応

**社会保険料**:
```
健康保険料 = staffs.health_insurance_amount（手動設定の固定額）
介護保険料 = staffs.care_insurance_amount（手動設定の固定額）
厚生年金 = staffs.pension_amount（手動設定の固定額）
雇用保険料 = (支給総額 - 交通費以外の非課税手当) × 0.005
  ※ 小数点以下は切り捨て（一般的な端数処理）
```

**その他控除**:
```
食事代 = 125円 × 食数（月次手動入力）
家賃 = staffs.rent_deduction（固定額）
```

**控除総額**:
```
控除総額 = 所得税 + 健康保険 + 介護保険 + 厚生年金 + 雇用保険 + 食事代 + 家賃
```

**差引支給額**:
```
差引支給額 = 支給総額 - 控除総額
```

### 賞与計算

- 支給額: 手動入力
- 所得税: 手動入力（自動計算なし）
- 社会保険料: 通常月と同じ控除設定を適用 or 手動上書き可
- 賞与明細として出力可能

### 現金封入用の紙幣・硬貨内訳計算

差引支給額から最小枚数の内訳を自動計算:
```
10,000円札: net_pay / 10000
 5,000円札: (残り) / 5000
 1,000円札: (残り) / 1000
   500円玉: (残り) / 500
   100円玉: (残り) / 100
    50円玉: (残り) / 50
    10円玉: (残り) / 10
     5円玉: (残り) / 5
     1円玉: 残り
```

## 注意事項・補足

### 法令対応
- 週の起算日は水曜日（店舗独自設定）
- 法定休日の考え方を適用（週1日の休日を確保する前提で6日目以降が割増）
- 割増率の重複: 加算しない（常に1.25倍）

### 運用上の注意
- 時給や社会保険料の変更は即時反映（過去月の計算には影響しない、計算確定済みのデータは保持）
- 源泉徴収税額表は年度ごとに更新が必要
- データ保存期間: 7年（法定保存義務に準拠）

### 将来の拡張候補（今回は実装しない）
- 住民税の控除
- 家族手当
- 前払金（仮払い）管理
- 有給休暇の管理（付与・消化・残日数）
- 有給買取
- シフト管理
- 昇給履歴管理
- 扶養人数の管理（所得税の甲欄で扶養人数別の税額適用）
